Using the JAXB XJC Maven 2 Plugin

Overview

The JAXB xjc plugin is a Maven version 2 plugin that invokes the xjc compiler. 

The Java Architecture for XML Binding (JAXB) provides a binding compiler, xjc, to generate Java classes from an XML Schema. The xjc binding compiles an XML schema into one or more Java classes. These automatically-generated classes handle the translation between XML documents that follow the schema and interrelated instances of the derived classes. They also ensure that the constraints expressed in the schema are maintained as instances of the classes are manipulated. The Java classes generated with the JAXB xjc plugin represent the different elements and complexTypes in an XML Schema. 

See references below for more information about JAXB, xjc and Maven.

Quick Start

If you already have Maven 2 installed and a valid schema then running this xjc compiler plugin will take just a few minutes.  Maven will automatically download and install the compiler and the necessary jaxb jars in your local repository.  The xjc plugin will compile your schemas during the standard Maven generate phase which occurs just before the compile phase.  The generated java files are placed in the Maven target directory.

To invoke the xjc compiler with Maven 2 just follow these steps.

1.      Place your schema files (*.xsd or *.dtd) into your src\main\resources directory.

2.      Merge into your Maven pom.xml file under <project> the following three (3) sections:

    <repositories>

        <repository>

            <id>java.net</id>

            <name>java.net Maven Repository</name>

            <url>https://maven-repository.dev.java.net/nonav/repository</url>

            <layout>legacy</layout>

        </repository>

    </repositories>

 

    <dependencies>

        <dependency>

            <groupId>jaxb</groupId>

            <artifactId>jaxb-api</artifactId>

            <version>2.0-SNAPSHOT</version>

        </dependency>

        <dependency>

            <groupId>jaxb</groupId>

            <artifactId>jaxb-impl</artifactId>

            <version>2.0-SNAPSHOT</version>

        </dependency>

    </dependencies>

 

    <build>

        <plugins>

            <plugin>

                <groupId>com.sun.tools.xjc.maven2</groupId>

                <artifactId>maven-jaxb-plugin</artifactId>

                <executions>

                    <execution>

                        <goals>

                            <goal>generate</goal>

                        </goals>

                    </execution>

                </executions>

            </plugin>

            <plugin>

                <groupId>org.apache.maven.plugins</groupId>

                <artifactId>maven-compiler-plugin</artifactId>

                <configuration>

                    <source>1.5</source>

                </configuration>

            </plugin>

        </plugins>

    </build>

3.    Invoke the compiler by just executing the standard compile, package or install Maven goals. 

That’s it.

The plugin and Maven will take care of the downloading and compilation details.  As Maven runs you will see messages from the xjc compiler and upon completion you will see the generated sources under target\generated-sources\xjc.

Configuration

The above example is the minimum configuration.  You can, of course, adjust all the default configurations to suite your file paths and xjc compilation needs.  For instance if you want xjc to generate all of the java files in a specific package just add a configuration section to the xjc plugin and define the generatePackage setting like this:

    <plugin>

        <groupId>com.sun.tools.xjc.maven2</groupId>

        <artifactId>maven-jaxb-plugin</artifactId>

        <executions>

            <execution>

                <goals>

                    <goal>generate</goal>

                </goals>

            </execution>

        </executions>

        <configuration>

            <generatePackage>org.example.package</generatePackage>

        </configuration>

    </plugin>

A more complex configuration may look similar to this.

    <plugin>

        <groupId>com.sun.tools.xjc.maven2</groupId>

        <artifactId>maven-jaxb-plugin</artifactId>

        <executions>

            <execution>

                <goals>

                    <goal>generate</goal>

                </goals>

            </execution>

        </executions>

        <configuration>

            <generatePackage>org.example.package</generatePackage>

            <includeSchemas>

                <includeSchema>*.xsd</includeSchema>

                <includeSchema>*.dtd</includeSchema>

            </includeSchemas>

            <excludeSchemas>

                <excludeSchema>test*.xsd</excludeSchema>

            </excludeSchemas>

            <includeBindings>

                <includeSchema>*.xjb</includeSchema>

            </includeSchemas>

            <strict>true</strict>

            <verbose>true</verbose>       

        </configuration>
    </plugin>

 

Configuration Properties

This is a complete list of the JAXB XJC Maven 2 Plugin settings.  These settings all appear in the plugin <configuration> section.  All settings are optional.

Property

schemaDirectory

Description

The source directory containing *.xsd schema and *.xjb binding files.

Type

File

Default

${basedir}/src/main/resources

Example

src/main/resources

 

Property

includeSchemas

Description

A list of regular expression file search patterns to specify the schemas to be processed.  Searching is based from the root of schemaDirectory. If this is not set then all .xsd files in schemaDirectory will be processed.

Type

String[]

Default

*.xsd

Example

src/main/resources

 

Property

excludeSchemas

Description

A list of regular expression file search patterns to specify the schemas to be excluded from the includeSchemas list.  Searching is based from the root of schemaDirectory.

Type   

String[]

Default

*.xsd

Example

src/main/resources

 

Property

includeBindings

Description

A list of regular expression file search patterns to specify the binding files to be processed.  Searching is based from the root of schemaDirectory.  If this is not set then all .xjb files in schemaDirectory will be processed.

Type

String[]

Default

*.xjb

Example

mybinding.xjb

 

Property

excludeBindings

Description

A list of regular expression file search patterns to specify the binding files to be excluded.  Searching is based from the root of schemaDirectory.

Type   

String[]

Default

*.xjb

Example

test*.xjb

 

Property

generatePackage

Description

If specified, generated code will be placed under this Java package.

Type   

String

Default

[none]

Example

org.some.package

 

Property

generateDirectory

Description

Generated code will be written under this directory. If you specify target="doe/ray" and generatePackage="org.here", then files are generated to doe/ray/org/here.

Type   

String

Default

${project.build.directory}/generated-sources/xjc

Example

org.some.package

 

Property

readOnly

Description

Generate Java source files in the read-only mode if true is specified.

Type

Boolean

Default

false

Example

true or false

 

Property

extension

Description

If set to true, the XJC binding compiler will run in the extension mode. Otherwise, it will run in the strict conformance mode.

Type   

Boolean

Default

false

Example

true or false

 

Property

catalog

Description

Specify the catalog file to resolve external entity references. Support TR9401, XCatalog, and OASIS XML Catalog format.

Type   

File

Default

[none]

Example

 

 

Property

removeOldOutput

Description

Used in pair with nested <produces> elements. When this attribute is specified as "true", the files pointed to by the <produces> elements will be all deleted before the XJC binding compiler recompiles the source files.

Type

Boolean

Default

false

Example

true or false

 

Property

strict

Description

Perform strict validation of the input schema.

Type   

Boolean

Default

true

Example

true or false

 

Property

verbose

Description

If verbose all the configured settings that are to be passed to the xjc compiler are logged.

Type      

Boolean

Default

false

Example

true or false
